实体名称规则

Go 中的函数、变量、常量、类型、预计标签和包的名称遵循一个简单的规则：名称的开头是一个字母（Unicode中的字符即可）或者下划线，后面可以跟任意数量的字符、数字和下划线，并且区分大小写。你要是愿意，用中文变量都可以：

如果一个实体在函数中声明，它只有在函数局部有效。如果声明在函数外，它将对包里面的所有源文件可见。实体名称的第一个字母的大小写决定它是否可跨包访问。如果名称是以大写字母开头，意味着它是可导出的，对包外是可见和可访问的，可以被自己包之外的程序所引用，像fmt中的Printf。包的名称总是由小写字母组成。

声明

变量声明

声明就是给一个程序实体命名，并且设定其部分或者全部属性。有4个主要声明：变量(var)声明，常量(cont)声明，类型(type)声明，函数(func)声明。

Go 程序实体存储在一个或者多个以.go为后缀的我呢间里。每一个文件以package开头，表明属于哪个宝。package声明后面就是import声明，然后包级别的类型，变量，函数，常量声明，并且不分顺序。要注意的是：包级别的实体不仅对于包含其声明的源文件可见，而且对于同一个包里面的所有源文件都可见。另一方面，局部声明仅仅是在所在声明函数内部可见，并且可能是函数中的一块区域

实参是按值传递的，所以函数接受到的是每个实参的副本，修改形参不会影响调用者提供的实参。然而，如果提供的实参包含引用类型，比如指针，slice，map，函数或者通道，那么当函数使用形参变量时有可能简介修改实参。

指针

变量是存储值的地方，而指针的值是一个变量的地址。一个指针指示值保存的位置，不是所有的值都有地址，但是所有的变量都有。使用指针，可以在无须知道变量名字的情况下，间接读取或者更新变量的值。

例如，如果声明一个变量：var x int，表达式 &x 获取一个指向整形变量的指针，它的类型是整形指针*int。如果值叫做p，可以说p指向x，或者说p包含x的地址，p指向的变量写成*p，表达式*p获取变量的值，代表一个变量，所以它也可以出现在复制表达式的左边。

指针的零值是nil, 测试 p != nil，结果是true，说明 p 指向一个变量。指针是可以比较的，当前仅当两个指针指向同一个变量或者都为nil才相等。

变量的生命周期

生命周期指的是在程序执行过程中，变量存在的时间段。包级别变量的生命周期是整个程序的执行时间。但是局部变量有一个动态的声明周期：每次执行声明语句的时候创建一个新的实体，变量一直生存到它变得不可访问，这个时候他占用的内存空间被回收。

垃圾回收器如何确定一个变量是否该回收了？基本思路是这样的：一个包级别的变量，以及每一个当前执行函数的局部变量，可以作为追溯该变量的路径的源头，通过指针或者其他方式的引用可以找到该变量。如果变量的路径不存在，那么这个变量将变得不可访问，它就会被垃圾回收器回收，也不会影响任何其他的计算过程。

因为变量的声明周期是通过它是否可达来确定的，所以局部变量可在包含它的循环的一次迭代之外继续存活。即使它的循环已经返回，它的存在还可能延续。

编译器可以选择使用堆或栈上的空间来分配内存，但是这个并不基于使用 var 还是 new。

整数

Go 同时具备有符号整数和无符号整数，分为四种大小：8位，16位，32位，64位，其中有符号为：int8, int16, int32, int64，无符号为：uint8, uint16, uint32, uint64。此外还有两种类型：uint 和 int。在特定平台上，它与原生的有符号整数、无符号整数相同，等于该平台上运行效率最高的值，它俩的大小相同，都是32位或者64位，但不一定就是32或者64。rune 类型是 int32 的同义词，常常用于指明值是一个Unicode码点，这两个名称可以互换使用，同样，byte类型是 uint 类型的同义词，强调一个值是原始数据，而非量值。最后，还有一种无符号整数 uintptr，其大小不明确，但足以完整存放指针，uintptr 类型仅仅用于底层编程，

int, uint 和 uintptr 都有别于其大小明确的相似类型的类型。就是说，int 和 int32 是不同的类型，尽管 int 的天然大小是 32位，但是 int 的值若要当 int32 使用，必须显示转换。

对于Go而言，取模余数的正负号都与被除数一致，例如 -5 % 3 和 -5 % -3 都是 -2。

对于一个整数，能写成十进制，就也能写成八进制，以0开头，比如0666；还能写成16进制数，以 0x 或者 oX 开头，如： 0xdefadeeef。当前八进制数仅有一种用途，表示POSIX文件系统的权限;
