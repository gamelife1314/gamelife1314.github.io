### 实体名称规则

Go 中的函数、变量、常量、类型、预计标签和包的名称遵循一个简单的规则：名称的开头是一个字母（Unicode中的字符即可）或者下划线，后面可以跟任意数量的字符、数字和下划线，并且**区分大小写**。你要是愿意，用中文变量都可以：

```go
package main

import "fmt"

func main() {
	var 变量 = "name"
	fmt.Println(变量)  // 这里完全ok的，可以编译通过并且运行，只是这么做有点别扭
}
```

如果一个实体在函数中声明，它只有在函数局部有效。如果声明在函数外，它将对包里面的所有源文件可见。实体名称的第一个字母的大小写决定它是否可跨包访问。如果名称是以大写字母开头，意味着它是可导出的，对包外是可见和可访问的，可以被自己包之外的程序所引用，像`fmt`中的`Printf`。**包的名称总是由小写字母组成。**

### 声明

#### 变量声明

声明就是给一个程序实体命名，并且设定其部分或者全部属性。有4个主要声明：变量(`var`)声明，常量(`cont`)声明，类型(`type`)声明，函数(`func`)声明。

Go 程序实体存储在一个或者多个以`.go`为后缀的我呢间里。每一个文件以`package`开头，表明属于哪个宝。`package`声明后面就是`import`声明，然后包级别的类型，变量，函数，常量声明，并且不分顺序。要注意的是：**包级别的实体不仅对于包含其声明的源文件可见，而且对于同一个包里面的所有源文件都可见。另一方面，局部声明仅仅是在所在声明函数内部可见，并且可能是函数中的一块区域**

#### 常量声明

常量是一种表达式，其可以保证在编译阶段就计算出表达式的值，并且不需要等到运行时。所有常量基本上都属于基本类型：布尔型，字符串或数字；常量声明定义了具名的值，他看起来语法上与变量类似，但该值恒定，这防止了程序运行过程中的意外（或恶意）修改。常量声明可以同时指定类型和值，如果没有显示指定类型，则根据右边的表达式进行推断。

```go
package main

import (
	"fmt"
	"time"
)

const noDelay time.Duration = 0

func main() {
	fmt.Printf("%T %[1]v", noDelay) // time.Duration 0s
```

如果同时声明一组常量，除了第一项之外，其他在等号右侧的表达式都是可以省略的，这意味着会复用前面一项的表达式及其类型，例如：

```go
package main
import (
	"fmt"
)
const (
	a = 1
	b
	c = 2
	d
)
func main() {
	fmt.Println(a, b, c, d)  // 1 1 2 2 
}
```

常量的声明可以使用常量生成器 `iota`， 它创建一系列相关的值，而不是逐个值显示写出，**常量声明中，`iota`从`0`开始取值，逐项加`1`**。

```go
package main

import (
	"fmt"
)

type Weekday int

const (
	Sunday Weekday = iota
	Monday
	Tuesday
	Wednesday
	Thursday
	Friday
	Saturday
)

func main() {
	fmt.Println(Sunday, Monday, Tuesday, Wednesday, Thursday, Friday, Saturday)  // 0 1 2 3 4 5 6

}
```

按照上面说的，常量可以不赋值，那么他会复制上面的，所以`Monday`复制`Subday`的值`iota`，但是`iota`此时已经变为1，剩下的以此类推。也可以进行一些复杂的声明，例如：

```go
package main

import (
	"fmt"
)

const (
	_ = 1 << (10 * iota)
	KB
	MB
	GB
	TB
	PB
	EB
	ZB
	YB
)

func main() {
	fmt.Println(KB, MB, GB, TB, PB, EB) // 1024 1048576 1073741824 1099511627776 1125899906842624 1152921504606846976
	fmt.Println(YB / ZB) // 1024
}
```

上面的ZB，和YB已经远远超出Go类型存储的范围，但是他们依然是有效的常量，这是因为编译器将这些从属类型待定的常量表示成某些值，这些值比基本类型的数字精度更高，且算数精度高于原生的机器精度，可以认为他们的精度至少达到256位。从属类型待定的常量一共有6种，分别是：无类型布尔，无类型整数，无类型文字符号，无类型浮点数，无类型复数，无类型字符串。借助推迟确定从属类型，无类型常量不仅能暂时维持更高的精度，与类型已经确定的常量相比，他们还能写进更多表达式而无需转换类型。

#### 函数声明

每个函数都有一个名字，一个形参列表，一个可选的返回列表以及函数体：

```
func name(param-list) (return-list) {
    body
}
```

形参的列表指定了一组变量的参数名和参数类型，这些局部变量都由调用者提供的实参传递而来。返回列表则制定了返回值的类型。**当函数返回一个未命名的返回值或者没有返回值的时候，返回列表的圆括号可以省略**。如果一个函数没有返回值，那么设计这个函数的目的就是调用之后的附加效果。

```go
func hypot(x, y float64) float64 {
    return math.Sqrt(x*x + y*y)
}

fmt.Println(hypot(3, 4))
```

x, y是函数形参，3，4是函数调用的实参，并且返回一个类型为 `float64` 的值。**返回值也可以像形参一样命名，这个时候，每一个命名的返回值会声明为一个局部变量，并且根据变量类型初始化为响应的零值**。当函数存在返回值的时候，函数必须明确以`return`语句结束，除非确定函数不会走完整个执行流程。

上面的函数中使用到一种简写，如果几个参数或者返回值的类型相同，那么类型只需要写一次，所以下面的两个生命完全是相同的：

```
func(i, j, k int, s, t string) {}
func(i int, j int, k int, s string, t string) {}
```

函数的类型称作**函数签名**，当两个函数拥有相同的形参列表以及返回列表时，认为这两个函数的签名相同。而形参和返回值的名字不会影响函数的类型。

需要谨记的是：**Go语言中没有默认参数这个概念，也不能指定实参名**，所以每一次调用函数都需要提供实参来对应函数每一个形参。形参列表和命名返回值都属于函数最外层作用域的局部变量。

实参是按值传递的，所以函数接受到的是每个实参的副本，修改形参不会影响调用者提供的实参。然而，如果提供的实参包含引用类型，比如指针，slice，map，函数或者通道，那么当函数使用形参变量时有可能简介修改实参。


### 指针

变量是存储值的地方，而**指针**的值是一个变量的地址。**一个指针指示值保存的位置，不是所有的值都有地址，但是所有的变量都有**。使用指针，可以在无须知道变量名字的情况下，间接读取或者更新变量的值。

例如，如果声明一个变量：`var x int`，表达式 `&x` 获取一个指向整形变量的指针，它的类型是整形指针`*int`。如果值叫做`p`，可以说`p`指向`x`，或者说`p`包含`x`的地址，`p`指向的变量写成`*p`，表达式`*p`获取变量的值，代表一个变量，所以它也可以出现在复制表达式的左边。

```go
package main

import (
	"fmt"
)

func main() {
	x := 1
	p := &x
	fmt.Printf("%T %[1]d；%T %[2]X；%T %[3]d；", x, p, *p) // int 1；*int C42001C080；int 1；
	*p = 2
	fmt.Println(x, *p) // 2 2
}

```

指针的零值是`nil`, 测试 `p != nil`，结果是`true`，说明 `p` 指向一个变量。指针是可以比较的，当前仅当两个指针指向同一个变量或者都为`nil`才相等。

函数返回局部变量的地址是非常安全的。如下函数：

```go
func f() *int {
	num := 2
	return &num
}

func main() {
	p := f()
	*p = 3
	fmt.Println(*p) // 3
}
```

因为一个指针包含一个变量的地址，所以传递一个指针参数给函数，能够让函数更新简介传递的变量值。

```go
func f(p *int) int {
	*p++
	return *p
}

func main() {
	num := 3
	f(&num)
	fmt.Println(num)  //4
}
```

### new 函数

可以使用内置的 `new` 函数创建变量，表达式`new(T)`创建一个未命名的`T`类型变量，并且初始化为`T`类型的零值，然后返回其地址。

```go
num := new(int)
fmt.Printf("%T %[1]v %T %[2]d", num, *num) // *int 0xc42001c080 int 0
```

使用`new`创建的变量和取其地址创建的普通局部变量没什么不同，只是不需要引入和声明一个虚拟的名字，通过`new(T)`就可以直接在表达式中使用，因此 `new` 只是语法上的便利。每一次调用 `new` 都会返回一个具有唯一地址的不同变量：

```go
num := new(int)
num1 := new(int)
fmt.Println(num == num1) // false
```

### 变量的生命周期

生命周期指的是在程序执行过程中，变量存在的时间段。包级别变量的生命周期是整个程序的执行时间。但是局部变量有一个动态的声明周期：每次执行声明语句的时候创建一个新的实体，变量一直生存到它变得不可访问，这个时候他占用的内存空间被回收。

垃圾回收器如何确定一个变量是否该回收了？基本思路是这样的：一个包级别的变量，以及每一个当前执行函数的局部变量，可以作为追溯该变量的路径的源头，通过指针或者其他方式的引用可以找到该变量。如果变量的路径不存在，那么这个变量将变得不可访问，它就会被垃圾回收器回收，也不会影响任何其他的计算过程。

因为变量的声明周期是通过它是否可达来确定的，所以局部变量可在包含它的循环的一次迭代之外继续存活。即使它的循环已经返回，它的存在还可能延续。

编译器可以选择使用堆或栈上的空间来分配内存，但是这个并不基于使用 `var` 还是 `new`。

```go
var global *int

func f() {
	x := 1
	global = &x
}

func main() {
	f()
	fmt.Println(*global) // 1
}
```

这里的 `x` 一定使用堆空间，因为它在`f`函数执行结束之后还可以从 `global` 访问，尽管它被声明为一个局部变量，这种情况我们说 `x` 从 `f` 中逃逸。

```go
func g() {
	y := new(int)
	*y = 1
}
```

但是 `g` 函数返回时，变量 `*y` 变得不可访问，可回收，因为 `*y` 并没有从 `g` 中逃逸。

### 包初始化

包的初始化从包级别的变量开始，这些变量按照声明顺序初始化，在依赖解析完毕的情况下，根据依赖的顺序进行。

如果包由多个 `.go` 文件组成，初始化按照编译器收到文件的顺序进行：go 工具会在调用编译器前将 `.go` 文件进行排序。对于包级别的变量，生命周期从其值被初始化开始，但是对于其它一些变量，比如数据表，初始化表达式不是简单地设置它的初始化值。这种情况下，init 函数的机制会比较简单。**任何文件**可以包含**任意数量**的声明如下的函数：

```
func init() {}
```

这个 `init` 函数不能被调用和引用，另一方便，它是普通的函数。在每一个文件里，当程序启动的时候，`init` 函数会按照它们声明的顺序自动执行。

```go
package main

import "fmt"

func f() {
	y := new(int)
	*y = 1
}

func init() {
	fmt.Println("hello")
}

func init() {
	fmt.Println("world")
}

func main() {
	fmt.Println("Go")
}

```

输出如下：

    hello
    world
    Go

包的初始化顺序按照在程序中导入的顺序来进行，依赖顺序优先，每次初始化一个包。因此如果 包`p` 导入了 包 `q`, 可以确保 `q` 在 `p` 之前完成初始化。初始化的过程是自下向上的，`main` 包最后初始化。 

### 整数

Go 同时具备有符号整数和无符号整数，分为四种大小：8位，16位，32位，64位，其中有符号为：`int8`, `int16`, `int32`, `int64`，无符号为：`uint8`, `uint16`, `uint32`, `uint64`。此外还有两种类型：`uint` 和 `int`。在特定平台上，它与原生的有符号整数、无符号整数相同，等于该平台上运行效率最高的值，它俩的大小相同，都是32位或者64位，但不一定就是32或者64。`rune` 类型是 `int32` 的同义词，常常用于指明值是一个Unicode码点，这两个名称可以互换使用，同样，byte类型是 uint 类型的同义词，强调一个值是原始数据，而非量值。最后，还有一种无符号整数 `uintptr`，其大小不明确，但足以完整存放指针，`uintptr` 类型仅仅用于底层编程，

`int`, `uint` 和 `uintptr` 都有别于其大小明确的相似类型的类型。就是说，`int` 和 `int32` 是不同的类型，尽管 `int` 的天然大小是 32位，但是 `int` 的值若要当 `int32` 使用，必须显示转换。

**对于Go而言，取模余数的正负号都与被除数一致**，例如 `-5 % 3 ` 和 `-5 % -3` 都是 `-2`。

对于一个整数，能写成十进制，就也能写成八进制，以0开头，比如0666；还能写成16进制数，以 `0x` 或者 `oX` 开头，如： `0xdefadeeef`。当前八进制数仅有一种用途，表示POSIX文件系统的权限;

### 字符串

字符串是不可变的字节序列，它可以包含任意数据，包括0值字节，但主要是人类可读的文本。内置的 `len` 函数会返回字符串的字节数而非文字符号的数目，下表访问操作`s[i]`则可以获得第i个字符,其中 `0 <= i <= len(s)`。

子串生成操作s[i:j]产生一个新子串，内容取自原字符串的字节，区间为：`[i, j)`，结果大小是 `j - i` 个字节。

`+` 运算符连接两个新字符串而生成一个新的字符串；

尽管肯定可以将新值赋予字符串变量，但是字符串值无法改变：字符串值本身包含的字节序列永不可变，如果要在一个字符串后面添加另一个字符串，可以这样做：

```go
func main() {
	s := "left foot"
	t := s
	s += ", right food"
	fmt.Printf("%s \n %s", t, s)  
}
```

输出：

    left foot 
    left foot, right food

因为字符串不可改变，所以字符串内部的数据不允许修改：`s[0] = 'L'`，将会编译错误。

字符串的值可以直接写成字符串字面量，形式上就是带引号的字节序列：`"hello，世界"`，因为 Go 的源码文件总是按 UTF-8 编码，并且习惯上 Go 的字符串会按 UTF-8 解读，所以在源码中我们可以将 Unicode 码点写入字符串字面量。

原生字符串字面量写法是：`` `...` ``，使用反引号而不是双引号。原生字符串字面量内，转移序列不齐作用，实质内容与字面写法严格一致，包括反斜杠和换行符，因此，在程序源码中，原生的字符串字面量可以展开多行。唯一特殊处理的是回车符会被删除（换行符会被保留）。

UTF-8 字符串长度：

```go
package main

import (
	"fmt"
	"unicode/utf8"
)

func main() {
	s := "hello，中国"
	fmt.Printf("len(s) 的长度是: %d\n", len(s))
	fmt.Printf("s 的实际长度是: %d\n", utf8.RuneCount([]byte(s)))
}
```

输出如下：

    len(s) 的长度是: 14
    s 的实际长度是: 8

按UTF-8解码处理字符串：

```go
package main

import (
	"fmt"
	"unicode/utf8"
)

func main() {
	s := "hello，中国"
	for i := 0; i < len(s); {
		r, size := utf8.DecodeRuneInString(s[i:])
		fmt.Printf("%d\t%c\n", i, r)
		i += size
	}
}
```

输出如下：

        0	h
        1	e
        2	l
        3	l
        4	o
        5	，
        8	中
        11	国

使用Go的`range`循环遍历字符串，它会对字符串进行UTF8隐式解码：

```go
package main

import (
	"fmt"
)

func main() {
	s := "hello，中国"
	for i, c := range s {
		fmt.Printf("s[%d]=%c %[2]T\n", i, c)
	}
}
```

输出如下：

        s[0]=h int32
        s[1]=e int32
        s[2]=l int32
        s[3]=l int32
        s[4]=o int32
        s[5]=， int32
        s[8]=中 int32
        s[11]=国 int32

使用 `bytes.Buffer` 增量写入字符串：

```go
package main

import (
	"bytes"
	"fmt"
)

func ints2String(values []int) string {
	var buf bytes.Buffer
	buf.WriteByte('[')
	for i, v := range values {
		if i > 0 {
			buf.WriteString(", ")
		}
		fmt.Fprintf(&buf, "%d", v)
	}
	buf.WriteByte(']')
	return buf.String()
}

func main() {
	ints := []int{1, 2, 3}
	fmt.Println(ints2String(ints))  // [1, 2, 3]
}

```

如果要在 bytes.Buffer 变量后面增加任意文字符号的utf-8编码，最好使用`bytes.Buffer` 的 `WriteRune`方法，而追加ASCII字符，使用 `WriteByte` 即可。
